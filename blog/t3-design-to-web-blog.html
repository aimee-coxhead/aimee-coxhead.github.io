Design to Web

What a responsive site is, and why responsiveness is important

A responsive site is a website that works on any size screen. There is one set of code, but the code changes how the website looks depending on the screen size. Responsiveness is important as people view websites from may different devices and you want the website to work well on different devices/ screen sizes.

What mobile first design is, and why it's important

Mobile first design is when you design your website for mobile devices first and then design for other (larger) devices after. The belief is that mobile design is harder and once this is sorted it will then be easier to design for other devices. The reason why it is important is because so many people access information on a mobile device and you need websites to be accessible and easy to use on these (smaller) devices.

What frameworks are, and their pros and cons

A framework is predefined set of concepts/components that can be used to help support the making of a website.

    Pros
  • Don’t have to do the code from scratch - Saves time
  • Clean and tidy code
  • Solutions to common CSS problems
  • Can be helpful when working with others
    Cons
  • You don’t learn how to do it yourself
  • Mixes content and presentation

What a wireframe is and why we use it

A wireframe is a sketch of a website before you make it. The reasons why you use a wireframe are:

  • You are able to see where the most important elements are going to be on the page.
  • It allows you to communicate clearly with others about where things will be located on the webpage.
  • It helps you outline where things go so that you can use it as a reference when creating your website.

The aspects of your wireframes you found difficult to implement, and why

I found moving the drop-down box to the center difficult. I really wanted to create this, so I looked on YouTube to find out how to do it. With the one I saw you had to make it ‘float: left’ which meant that when I tried to use ‘text-align: cente’ it wouldn’t work. I researched further and was able to come up with a way that I didn’t have to use 'float: left' and this meant I could use ‘text-align’ to center my drop-down box.

I also wanted to make the home button a picture of a house and put a little arrow going down after ‘blogs’ (so that people knew it was a drop-down bar). However, I was unsure how to do this and didn’t have the time, so I just left it out. I will try go back in the future and add these in.
